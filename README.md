# lark_ai_challenge_openclaw_longterm_memory

飞书 AI 挑战赛 - OpenClaw 企业级长程记忆方向。

## 项目目标

本仓库用于 **结合飞书 CLI 与 OpenClaw** 进行开发，形成一个可落地的企业级长程记忆方案：

- 使用飞书 CLI 完成应用创建、配置与本地调试；
- 使用 OpenClaw 承担长程记忆能力（记忆写入、召回与上下文增强）；
- 将飞书消息流与 OpenClaw 记忆流打通，支持持续对话与知识沉淀。

## 推荐开发流程

1. **飞书侧初始化（CLI）**
   - 使用飞书 CLI 创建/拉取应用配置；
   - 配置事件订阅、机器人权限与回调地址；
   - 在本地启动调试服务并验证事件回调可达。

2. **OpenClaw 侧初始化**
   - 部署或连接 OpenClaw 服务；
   - 配置记忆存储（如向量库/数据库）与检索策略；
   - 暴露记忆写入、检索与会话管理接口。

3. **联调集成**
   - 飞书消息进入后，先做会话归一化（tenant/user/thread）；
   - 调用 OpenClaw 检索历史记忆并增强 prompt；
   - 模型回复后将结构化结果回写 OpenClaw；
   - 在飞书侧返回最终响应并记录链路日志。

## 最小集成检查清单

- [ ] 飞书 CLI 本地调试可收到事件
- [ ] OpenClaw 记忆接口可正常读写
- [ ] 同一会话可复用历史记忆
- [ ] 重启服务后记忆可持久化召回
