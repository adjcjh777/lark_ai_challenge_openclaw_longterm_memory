## 先看这个

这份审计回答三个问题：

- **MVP 是否完成？** 已完成 demo / pre-production 本地闭环，可以演示、复现、评测。
- **是否接入飞书？** 已接入受控飞书测试群 live sandbox，并新增 limited workspace pilot；但还不是生产全量 Feishu workspace ingestion。
- **是否接入 OpenClaw 做完整产品形态测试？** 已有 OpenClaw `fmc_*` 工具、本地 Agent 调用、websocket staging、受控真实 DM allow-path、2026-05-04 workspace 记忆 bot 单聊回读和 demo/pre-production completion gate；但还不是生产级长期运行。

当前已经成型的是一条受治理的 Copilot 主线：真实飞书来源先进入 `memory.create_candidate` 和 review policy；低风险内容可以自动确认，重要、敏感或冲突内容交给 reviewer / owner；所有 confirm / reject / undo 仍走 `CopilotService` / `handle_tool_request()`。

2026-05-04 新增的重点是 workspace ingestion。当前采用 lark-cli-first pilot，已覆盖 Drive / Wiki / Docs / Sheets / Base/Bitable 的受控发现和类型路由，并有 registry skip / stale / failed / cursor、mixed-source 本地佐证、warm-path latency gate、真实 lark-cli fetch latency gate、真实 docx + Bitable 多来源 temp-DB gate、真实 chat log + workspace resource gate、真实同结论 gate、支持 Drive/Wiki 只读扩搜的真实样本匹配发现器、总 readiness gate 和项目 normal Sheet 只读证据 gate。公开模板类 normal Sheet 已在临时库读出 `lark_sheet` source 和 candidate；完整 lark-cli fetch / source render / candidate route / registry write 用时 12.747s。真实项目提交稿 + 任务看板 Base 已在临时 DB 中跑出 4 个 sources、10 个 candidates、0 failed，用时 8.356s。真实非 @ 群消息日志 + 项目 docx/Base 已在同一临时 DB 跑出 `feishu_message=1`、`document_feishu=1`、`lark_bitable=3`、workspace resource candidates 10、0 failed，用时 7.779s。早期广泛只读扩搜证明当前账号没有自然出现的项目 normal Sheet 和同结论样本；用户批准后创建受控 normal Sheet，并由一条真实 OpenClaw Feishu 群消息复述同一 durable fact。最新 same-fact finder 返回 `ok=true`、`same_fact_match_count=1`、`strict_same_conclusion_gate_passed=true`，final readiness gate 返回 `goal_complete=true`、`status=pass`、`failures=[]`。随后修复 OpenClaw `before_dispatch` 的 p2p DM 路由，并用 bot 单聊 `/recall Workspace ingestion readiness gate Sheet 样本` 证明该 workspace Sheet 记忆能通过 `fmc_memory_search` 回读，回复带 `lark_sheet` 证据和 `permission: allow / scope_access_granted` 审计。这关闭 controlled workspace-ingestion readiness 目标；仍不代表生产全量 workspace crawler、长期运行服务或生产 SLO。

本审计可以作为对账入口，但不要把 dry-run、replay、测试群 sandbox、live embedding gate、limited workspace pilot 写成 production live、全量 workspace ingestion 或 productized live。要宣称全量 workspace 接入，必须让 `scripts/check_workspace_productized_ingestion_readiness.py --require-productized-ready` 在非示例 evidence manifest 上通过。
